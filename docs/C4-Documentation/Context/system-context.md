## Personas

### Video Editor User

Interacts with the system through the Web GUI (React SPA served at `/gui`). Manages video libraries, creates projects, positions clips on timelines, applies effects and transitions, configures audio mixes, selects layout presets for multi-input composition, and initiates batch renders. Receives real-time feedback via WebSocket events displayed in the activity log and status bar.

### AI Agent (Claude)

Interacts with the system through the REST API. Discovers available operations via OpenAPI schema auto-generated by FastAPI. Configures effects using AI hints included in the effect catalog endpoint (`GET /api/v1/effects`). Builds edit sequences programmatically by calling timeline, composition, audio, and effects endpoints.

### API Client (CLI / Script)

Any HTTP client consuming the REST API directly for automation, integration testing, or scripting workflows. Uses the same endpoints as the AI Agent.

### WebSocket Events

All events are actively wired — emitted by routers during real API operations:

| Event | Description |
|-------|-------------|
| `HEARTBEAT` | Periodic keepalive (configurable interval) |
| `HEALTH_STATUS` | Health or readiness status changes |
| `SCAN_STARTED` | Directory scan job initiated |
| `SCAN_COMPLETED` | Directory scan finished (includes video count) |
| `PROJECT_CREATED` | New project created |
| `TIMELINE_UPDATED` | Timeline tracks or clips modified |
| `LAYOUT_APPLIED` | Layout preset applied to a project |
| `AUDIO_MIX_CHANGED` | Audio mix configuration updated |
| `TRANSITION_APPLIED` | Transition applied between adjacent clips |

## External Dependencies

### FFmpeg / FFprobe

| Property | Value |
|----------|-------|
| Role | Video encoding, thumbnail generation, metadata extraction |
| Interaction | Subprocess execution (sync and async) with 30s timeout for ffprobe |
| Required | Yes — readiness probe checks binary availability |
| Version | No specific version required; system PATH binary used |

FFmpeg is the rendering engine. The system builds FFmpeg command-line arguments and filter graph strings in Rust, then executes them via subprocess. All filter syntax is generated programmatically (never from raw user input) with sanitization for text content.

### SQLite

| Property | Value |
|----------|-------|
| Role | Persistent storage for all domain entities |
| Interaction | `aiosqlite` for async non-blocking access |
| Location | `data/stoat.db` (configurable via settings) |
| Extensions | FTS5 for full-text video search |

SQLite is embedded — no external database server required. Schema is initialized idempotently on application startup. The FTS5 extension enables prefix-matching search on video filenames and paths.

## Personas

### Video Editor User

Interacts with the system through the Web GUI (React SPA served at `/gui`). Manages video libraries, creates projects, positions clips on timelines, applies effects and transitions, configures audio mixes, selects layout presets for multi-input composition, previews edits via HLS streaming with theater mode, monitors proxy generation status, submits render jobs (with format/quality selection), monitors render progress in real-time, and initiates batch renders. Receives real-time feedback via WebSocket events displayed in the activity log and status bar.

### AI Agent (Claude)

Interacts with the system through the REST API. Discovers available operations via OpenAPI schema auto-generated by FastAPI. Configures effects using AI hints included in the effect catalog endpoint (`GET /api/v1/effects`). Builds edit sequences programmatically by calling timeline, composition, audio, and effects endpoints. Submits and monitors render jobs via the render API.

### API Client (CLI / Script)

Any HTTP client consuming the REST API directly for automation, integration testing, or scripting workflows. Uses the same endpoints as the AI Agent.

## External Dependencies

### FFmpeg / FFprobe

| Property | Value |
|----------|-------|
| Role | Video encoding, thumbnail generation, metadata extraction |
| Interaction | Subprocess execution (sync and async) with 30s timeout for ffprobe |
| Required | Yes — readiness probe checks binary availability |
| Version | No specific version required; system PATH binary used |

FFmpeg is the rendering engine. The system builds FFmpeg command-line arguments and filter graph strings in Rust, then executes them via subprocess. The render subsystem uses async subprocess execution with real-time progress parsing via Rust bindings. Graceful cancellation sends stdin 'q' (Windows-safe) rather than process signals. All filter syntax is generated programmatically (never from raw user input) with sanitization for text content.

### SQLite

| Property | Value |
|----------|-------|
| Role | Persistent storage for all domain entities |
| Interaction | `aiosqlite` for async non-blocking access |
| Location | `data/stoat.db` (configurable via settings) |
| Extensions | FTS5 for full-text video search |

SQLite is embedded — no external database server required. Schema is initialized idempotently on application startup. The FTS5 extension enables prefix-matching search on video filenames and paths.

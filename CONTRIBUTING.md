# Contributing to stoat-and-ferret

## Contribution Policy

snf is developed solely by the maintainer; external code contributions are not accepted, and PRs are generally closed without review.

## CLA Backstop

In the exceptional event a contribution is accepted at the maintainer's sole
discretion, it requires a signed CLA — see [CLA.md](./CLA.md); unsigned
contributions cannot be merged. The CLA Assistant bot enforces this on every PR.

## Signing Flow

If a contribution is solicited by the maintainer:

1. Contributor opens a PR.
2. The CLA Assistant bot comments with sign-now link.
3. Contributor signs via the bot UI tied to their GitHub identity.
4. The bot records the signature in its database and unblocks merge.

## No DCO

This project does not use a Developer Certificate of Origin (DCO) or require
`Signed-off-by:` footers. The CLA (above) is the sole contribution backstop.

## Bug Reports and Feature Requests

Open a GitHub issue. Issues are read but not guaranteed a response.
